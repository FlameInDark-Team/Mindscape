======= Mindscape Feature Map

Mindscape v1 · Feature Map

Current feature set after the data-model rewrite

This file maps the actual shipped pilot behavior, not the earlier anonymous mood-only concept.

Student features

  • Anonymous daily pulse with mood, stress, energy, sleep, connectedness, and optional reflection.
  • Weekly PHQ-2 + GAD-2 screener flow.
  • Safety-status prompt and crisis-safe copy.
  • Resource routing based on risk band and support preference.
  • Explicit opt-in support request with separate identity capture.

Admin features

  • Magic-link login.
  • Dashboard metrics for daily pulses, weekly completion, high-concern share, and support queue.
  • Cohort watchlist with participation and weekly coverage.
  • Cohort detail route.
  • Calendar event context and recent support requests.

Infrastructure features

  • Hono + TypeScript + HTMX runtime.
  • D1-ready schema documenting the production model.
  • In-memory seeded repository for local demo use.
  • Groq-ready reflection tagging with heuristic fallback.
  • Resend-ready magic-link delivery with preview fallback.

Deferred features

  • Passive sensing remains phase 2 only.
  • Turnstile, retention jobs, and D1 wiring remain production hardening tasks.
  • No black-box prediction model in v1.
  • No therapist marketplace or teletherapy scheduling layer in v1.
